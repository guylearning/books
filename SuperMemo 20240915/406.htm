Item #401: [True / False]: HTML elements always follow the pattern: an opening tag + content + a closing tag.

[True / False]: HTML elements always follow the pattern: an opening tag + content + a closing tag.
False. Void elements with only an opening tag, like <img> and <br>.

Item #402: HTML: What is a void element? (top)

HTML: What is a void element?
An HTML element with only an opening tag, no closing tag or content.

Item #403: HTML: What is the term for an element with no content and no closing tag? (top)

HTML: What is the term for an element with no content and no closing tag?
A void element.

Item #404: In HTML, why add a forward slash (/) in the opening tag of a void element? (top)

In HTML, why add a forward slash (/) in the opening tag of a void element?
For XML, XHTML compatibility or readability.

Item #405: HTML: Can a void element contain content between its tags? (top)

HTML: Can a void element contain content between its tags?
(Trick question) No. Void elements in HTML are self-closing and do not have end tags.
