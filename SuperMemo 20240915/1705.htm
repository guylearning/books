Item #1,686: HTML: In an element, what are the function of the width and height attributes?

HTML: In an <img> element, what are the function of the width and height attributes?
Specify image dimensions.

Item #1,687: HTML: Which unit is used for the `width` and `height` attributes in an element? (top)

HTML: Which unit is used for the `width` and `height` attributes in an <img> element?
Pixels (px) by default.

  src="images/dinosaur.jpg"
  alt="The head and torso of a dinosaur skeleton"

Item #1,690: [True / False]: Browsers fetch the HTML and images in a single HTTP(S) request. (top)

[True / False]: Browsers fetch the HTML and images in a single HTTP(S) request.
False. Separate requests.

Browsers fetch HTML first, then parse it to identify and request additional resources like images, CSS, and JavaScript files. Each resource typically requires its own HTTP(S) request.

Item #1,696: HTML: What is the layout shift issue? (top)

HTML: What is the layout shift issue?
Unexpected changes in webpage layout during loading.

Layout shift in HTML refers to unexpected changes in the position of web elements during page load or interaction. These shifts can disrupt user experience by causing content to move unexpectedly, leading to misclicks or readability issues.

Item #1,691: HTML: What rendering issue arises without specifying image dimensions? (top)

HTML: What rendering issue arises without specifying image dimensions?
Layout shift.

Without specified dimensions, the browser doesn't reserve space for the image, causing content to shift as the image loads. This disrupts user experience and layout stability.

Item #1,692: HTML: What practice prevents image layout shifts during rendering? (top)

HTML: What practice prevents image layout shifts during rendering?
Specify image dimensions (width & height).

Specifying image dimensions helps the browser allocate space before loading, preventing layout shifts and improving page load speed.

Item #1,693: [True / False]: HTML: Using the `width` and `height` attributes to resize images. (top)

[True / False]: HTML: Using the `width` and `height` attributes to resize images.
False. Use image editor or CSS instead.

HTML attributes like width and height only change the display size, not the actual file size, leading to potential performance issues. CSS provides more flexibility and better control over image presentation.

Item #1,694: [True / False]: HTML: In an element, it's a good practice to set `width` and `height` attributes even if the image size is correct. (top)

[True / False]: HTML: In an <img> element, it's a good practice to set `width` and `height` attributes even if the image size is correct.
True.

Specifying image dimensions helps the browser allocate space before loading, preventing layout shifts and improving page load speed.

Item #1,695: HTML: Best way to correct image size to fit the webpage layout: [edit before upload / use HTML attributes]? (top)

HTML: Best way to correct image size to fit the webpage layout: [edit before upload / use HTML attributes]?
Editing the image before uploading ensures optimal quality, reduces file size, and improves load time and bandwidth efficiency.

Item #1,697: HTML: In an element, the `width` and `height` attributes are [required / optional]. (top)

HTML: In an <img> element, the `width` and `height` attributes are [required / optional].
Optional, but specifying them helps prevent layout shifts and improves page rendering stability.
